# tate-align(仮)仕様書 v1.3 — 全角対応・縦揃え整形ツール

## 実装エージェントへの指示

- この仕様書が唯一の正。着手前に実装計画を提示し、承認を得てから書くこと
- フレームワーク・ビルドツール・外部ライブラリは使用禁止(素のHTML/CSS/JSのみ)
- 仕様から逸脱したくなったら、実装せずに理由を添えて提案すること
- §8のテストが全て緑になるまで完成としない

## 1. 概要

貼り付けたテキストの列を、全角=2・半角=1の表示幅(East Asian Width)を正しく計算して縦に揃えるWebツール。ブラウザ内で完結し、入力はサーバーに一切送信しない。GitHub Pagesで公開する静的1ページ。

差別化の核:①全角幅計算の正確さ、②区切り文字が既定値入りの編集可能な欄に一本化されている(空白1個から任意の記号まで、ユーザーが自分のルールに書き換えられる)、③埋め方式を半角スペース/タブから選べる。

## 2. 対象ユーザーと利用場面

等幅フォント環境で「縦を揃えろ」と言われる人。ソースコードのコメント欄・変数宣言・構造体・定数表・配列初期化、仕様書、メール、Excelから貼った表など。入力は常に「どこかからのコピペ」を想定する。

## 3. スコープ(v1)

- ファイル構成:`index.html` / `style.css` / `app.js` / `tests.html` / `tests.js` の5ファイル
- 画面は「整形元」「整形後」の2枠構成(§9)
- 区切り欄(本ツールの中心概念):カンマ区切りのトークン列で、既定値 `\s,\t` が最初から入力されており、ユーザーが自由に編集・削除・追加できる(§5.4)
- 埋め方式(揃え方)は2択のトグルボタン:半角スペース(既定)/ タブ(§6)
- 整形モード:自動判定(既定)/ コードモード / 表モード
- オプション:埋め方式、ギャップ幅(1〜4、既定:コード1・表2。スペース方式のみ有効)、タブ幅(既定4)、=揃えON/OFF(既定ON、コードモードのみ)、区切り欄、出力改行コード(自動/LF/CRLF)

### 非スコープ(v1ではやらない)

プロポーショナルフォント向けの全角スペース近似埋め / Markdown表整形 / 保存・共有・アカウント / 右揃え(数値列)/ ファイルのアップロード・ダウンロード(職場ではダウンロード自体が許可制のため対象外。今後も予定なし)。

## 4. 幅計算仕様

- `charWidth(codepoint)`:East Asian Width区分でFullwidth/Wideは2、それ以外は1。Ambiguous(①・※・±など)は2として扱う(日本語等幅フォント前提の設計判断)
- 主な幅2レンジ(自前テーブルで実装):U+1100–115F、U+2E80–A4CF(CJK・かな・全角スペースU+3000含む)、U+AC00–D7A3、U+F900–FAFF、U+FE30–FE4F、U+FF00–FF60、U+FFE0–FFE6、絵文字(U+1F300–1FAFF は近似で2)
- `displayWidth(str)`:コードポイント単位の合計(`for...of`でサロゲートペア対応)
- 既知の限界としてREADMEに明記:結合文字・異体字セレクタ・一部絵文字は環境により表示幅が異なる

## 5. 整形ロジック仕様

共通:行頭インデントは保持(行頭の空白・タブは区切りとして扱わない)。出力の行末空白は除去。空行はそのまま。引用符(`'` `"` `` ` ``)の内側にある区切り・コメント記号はすべて無視する(簡易走査でよい)。

### 5.1 コードモード(コメント揃え)

1. 入力のタブを先にタブ幅で空白展開する
2. 各行を「コード部/コメント部」に分割。区切りは引用符外の最初の `//` または `#`。コメント記号の直前に空白がなくてもよい(`int a=0;// x` も対象)
3. コメントを持つ行だけを対象に `max(displayWidth(rstrip(コード部)))` を求め、埋め方式に従って揃える(§6)。コメントのない行は変更しない
4. コードモードでは区切り欄は使用しない

### 5.2 =揃え(コードモード内オプション)

1. 対象は「代入の=」のみ:`=` の直前直後が `= < > ! + - * / % & | ^` のいずれでもない、行内最初の1個
2. `pad(左辺, max) + " = " + trimStart(右辺)`(この部分の埋めは常に半角スペース)。コメント揃えと併用時は=揃えを先に適用

### 5.3 表モード

1. 区切り欄の内容に従って各行を列分割する(§5.4)
2. 列ごとに `max(displayWidth(セル))` を計算し、最終列以外を埋め方式に従って揃えて連結(§6)
3. 列数が異なる行:存在する列だけ揃え、超過分はそのまま。行末の区切りで生じる末尾の空セルは無視

### 5.4 区切り欄(本ツールの中心)

- 欄にはカンマ区切りでトークンを列挙する。既定値は `\s,\t`。特殊表記は3つ:
  - `\s` = 連続1個以上の半角スペースをひとつの区切りとする(空白1個でも列が分かれる)
  - `\t` = タブ
  - `\,` = カンマそのもの
  - それ以外の文字列は文字通りのトークン(複数文字可:`;` `:` `/` `//` `=>` など。全角スペースも文字として指定可)
- 分割規則:行内の各出現位置(引用符外)で列分割する。`\s` と `\t` は区切り自体を除去し、文字トークンは左セルの末尾に残す。分割後の各セルは先頭の空白をトリムしてから揃えに使う
- 既定値は編集・削除・追加が自由。例:セル内に単一スペースを含む表(例:`New York`)が壊れる場合は `\s` を消す/構造体を揃えたければ `;` を足す——という運用を想定し、READMEにこの例を書く
- 「既定に戻す」ボタンを欄の横に置く

### 5.5 自動判定

行末コメント記号(`//` `#`)を持つ行が過半 → コードモード。それ以外 → 表モード。判定結果をUIに表示し、手動上書き可能。

## 6. 埋め方式(半角スペース / タブ)

### 6.1 半角スペース方式(既定)

各セルを `pad(セル, 列max)` で半角スペース埋めし、ギャップ幅分の半角スペースを挟んで次列を置く。次列の開始表示位置 = 列max + ギャップ。

### 6.2 タブ方式

- 揃え先 = 列maxより大きい最小のタブ幅の倍数 T(列maxがちょうど倍数のときも次の倍数まで進める)
- 各セルの後ろに `ceil((T − displayWidth(セル)) / タブ幅)` 本のタブを置く
- ギャップ設定はタブ方式では無効(UIでグレーアウト)
- UIに注意書きを常時表示:「タブ揃えは貼り付け先のタブ幅が異なると見た目が変わります(このツールのタブ幅設定を合わせてください)」

## 7. 改行コードの扱い

- 貼り付け時に改行コードを自動検出(CRLF / LF。混在時は多数決)し、UIに検出結果を表示
- 出力プルダウン「改行コード:自動(入力と同じ)/ LF / CRLF」。既定は自動
- 実装注意:textarea内部ではLFに正規化されるため、検出は `paste` イベントのクリップボードデータで行い状態として保持。検出不能(手打ち等)はLF扱い

## 8. 入出力例とテスト

### 例A:コードモード(スペース方式、ギャップ1、=揃えOFF)

入力:

```
int a = 0; // 速度
int speed = 10; // 加速度の設定
long t = 3; // 時間
```

期待出力(`//` が全行同一桁):

```
int a = 0;      // 速度
int speed = 10; // 加速度の設定
long t = 3;     // 時間
```

### 例B:表モード(タブ区切り入力、区切り欄=既定、スペース方式、ギャップ2)

入力(セル間はタブ):

```
名前	点数	備考
田中	90	よい
Alexander	100	tops
```

期待出力(第2列が表示幅11、第3列が表示幅17から開始):

```
名前       点数  備考
田中       90    よい
Alexander  100   tops
```

### 例C:=揃え(コードモード)

入力:

```
speed = 0
加速度 = 10
name = "テスト"
```

期待出力(`=` が全行同一桁):

```
speed  = 0
加速度 = 10
name   = "テスト"
```

### 例D:区切り欄 `\s,\t,;`(既定に `;` を追加。コメント前に空白なし、表モード、ギャップ1)

入力:

```
int speed;// 速度
long acceleration;// 加速度
char c;// 文字
```

期待出力(`;` の後と `//` の後で列が分かれ、すべて揃う):

```
int  speed;        // 速度
long acceleration; // 加速度
char c;            // 文字
```

※ `// 速度` 内の空白でも分割されるが、`//`列・コメント本文列として自然に揃う。

### 例E:区切り欄 `\s,\t,\,`(配列初期化、表モード、ギャップ1)

入力:

```
{1,22,333},
{444,5,66},
```

期待出力(各カンマ位置で列が揃う。行末の空セルは無視):

```
{1,   22, 333},
{444, 5,  66},
```

### 例F:タブ方式(例Aと同じ入力、タブ幅4)

コード部の表示幅は 10 / 15 / 11、列max=15 → T=16。期待:1行目タブ2本、2行目タブ1本、3行目タブ2本が挿入され、タブ幅4の環境で `//` が全行表示幅16から開始(テストではタブ本数と展開後表示位置の両方を検証)。

### 例G:区切り欄=既定 `\s,\t`(空白1個の宣言が揃う、表モード、ギャップ1)

入力:

```
int a = 0;
int speed = 10;
long t = 3;
```

期待出力(単語ごとに列が揃い、`=` も自然に縦に並ぶ):

```
int  a     = 0;
int  speed = 10;
long t     = 3;
```

### テスト一覧(tests.html を開くと結果一覧が出る自前ランナー)

① `displayWidth`:ASCII/ひらがな/漢字/全角記号/①などAmbiguous/絵文字/サロゲートペア混在で期待値一致(8ケース以上)。
② 例A〜Gの一致(タブ方式はタブ本数+展開位置)。
③ エッジ:空行混在、コメントなし行混在、文字列リテラル内の`//`や`;`や空白で分割しない、`==`を=揃え対象にしない、列数不一致、行頭インデント保持(行頭空白で分割しない)、行末空白除去、連続スペースを1区切りに集約、`\s`を欄から消した場合は空白で分割しない、列maxがタブ幅の倍数ちょうどのときのタブ方式、CRLF入力→自動でCRLF出力・LF強制でLF出力、「既定に戻す」ボタン。

## 9. UI仕様

- 上部:タイトル+「全角が混ざってもズレない縦揃え」+「入力はブラウザ内でのみ処理されます(サーバー送信なし)」
- 本体:左右2枠(スマホは上下)——左「整形元」textarea(等幅、プレースホルダに例A)/右「整形後」(読み取り専用、等幅)+「コピー」ボタン(`navigator.clipboard`、成功フィードバック)
- 枠の間にオプション列:モード / 埋め方式トグル(スペース|タブ)/ ギャップ / タブ幅 / =揃え / 区切り欄(既定値 `\s,\t` 入り+「既定に戻す」ボタン) / 改行コード
- 入力・オプション変更のたびに即時整形。検出改行コードとモード判定結果を整形後枠付近に小さく表示。タブ方式選択時は§6.2の注意書きを表示
- `prefers-color-scheme`でダーク追従。フッタにGitHubリンク
